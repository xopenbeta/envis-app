Rust 中的 Command::output() 会阻塞当前线程直到执行的进程结束，并且它会等待子进程相关的标准输出流(stdout)和标准错误流(stderr)完全关闭后才返回。

当 Nginx 启动时，它会作为一个守护进程(后台服务)运行并派生出 Worker 子进程。问题在于，在 Windows 等操作系统下，派生出的后台子进程会继承父进程（即你用 Command 启动的那个进程）的 stdout/stderr 句柄。 因此，虽然 Nginx 的主进程可能很快就返回了，但它的子进程依然持续持有这些流通道。这导致 output() 方法一直在傻傻地等待缓冲区触发 EOF，而此时 Nginx 正处于正常的后台运行状态，从而导致 Rust 后端线程永久挂起，前端的 await startServiceData 也随之一同卡死。

* nginx的输出都放到日志里，命令这里独立运行，不捕获它的输出流（或者将输出流丢弃）。
