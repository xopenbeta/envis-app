## [0.13.6] - 2026-05-13

### 新增 / Added
- ✨ 新增服务运行状态自动推送机制：后端启动专用轮询线程（500ms 间隔），扫描所有已激活环境中的已激活服务，通过各服务管理器检测进程运行状态，一旦状态发生变化即主动向前端推送 `status:service` 事件，实现服务状态实时联动，无需前端主动轮询

### 改进 / Improved
- ♻️ 移除各服务面板（MariaDB / MongoDB / MySQL / Nginx / PostgreSQL / Redis）在启动 / 停止 / 重启操作后手动调用 `refreshServiceStatus()` 的逻辑，状态更新统一由后端事件驱动，降低前端冗余轮询开销
